# 03_Clock_Domain — 时钟域处理

## 为什么时钟域处理如此重要？

FPGA 芯片通常有多个时钟：系统时钟、外设时钟、音频时钟等。当数据跨越不同时钟域时，如果不做处理，会产生 **亚稳态（Metastability）**。

### 亚稳态

```
发送端时钟域 (clk_a)          接收端时钟域 (clk_b)
         │                          │
    data ──┤   ╔═══════╗             │
           │   ║ SETUP ║             │
           │   ║   <   ║  不满足!    │
           └──→║ HOLD  ║──────→ ???
               ╚═══════╝        │
                                ▼
                          ┌──────────┐
                          │  FF 输出  │
                          │ 不稳定!   │
                          │ 0? 1? X? │
                          └──────────┘
                                │
                          传播到后续逻辑
                          → 系统错误!
```

**亚稳态**发生在触发器的建立时间（setup time）或保持时间（hold time）被违反时。FF 输出既不是 0 也不是 1，而是中间电压，然后缓慢衰减到 0 或 1（耗时不确定）。

## 解决方案: 2 级同步器

```
发送域 (clk_a)              接收域 (clk_b)
                             ┌───┐   ┌───┐
  data_async ──────────────→│FF1│──→│FF2│──→ data_sync (安全!)
                             └───┘   └───┘
                               ↑       ↑
                             clk_b   clk_b
                             (异步采样) (已经稳定)
```

**原理**: 第一级 FF 可能亚稳态，但经过一个 clk_b 周期后，第二级 FF 采样时，第一级的输出大概率已经稳定。统计上，MTBF（平均无故障时间）随同步级数指数增长。

> **经验法则**: 2 级 FF 同步器足够应对绝大多数场景。极高速 (>500MHz) 可能需要 3 级。

## 不能同步的数据

| 数据类型 | 能否用 2 级 FF 同步 | 替代方案 |
|----------|---------------------|----------|
| 单比特信号 | ✅ | 2 级 FF |
| 多比特数据 | ❌ (各位可能不同步) | 异步 FIFO + 握手 |
| 总线数据 | ❌ | FIFO / 暂存寄存器 |
| 脉冲 (慢→快) | ✅ | 展宽 + 同步 |
| 脉冲 (快→慢) | ✅ | 互锁握手 |
