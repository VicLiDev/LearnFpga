# 02_Sequential_Logic — 时序逻辑

## 什么是时序逻辑？

时序逻辑的输出取决于**当前输入 + 历史状态**，需要时钟驱动，具有记忆功能。

```
             ┌──────────┐
Input ──→ ──→│ 组合逻辑  │──→ ──→ Output
             │          │
             └────┬─────┘
                  │
                  ▼
            ┌──────────┐
      ┌────←─│  D触发器  │←──── 时钟 (clk)
      │      └──────────┘
      │          │
      └──────────┘
        状态反馈
```

## 触发器 (Flip-Flop) — 时序逻辑的基础

### D 触发器
最基本的时序元件，每个时钟上升沿采样输入 D，传送到输出 Q。

```
        clk ──→ ┌───┐
                  │   │
  D ─────────────→│ D │
                  │   │──────→ Q (输出)
           ┌──────│>  │
           │      │   │
           │      └───┘
           │
           └── Q' (反相输出, 可选)
```

**行为**: 在时钟上升沿，Q = D（即 Q 在下一个时钟周期变成 D 的当前值）

## 时序逻辑的 always 块模板

```verilog
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        // 复位: 所有寄存器赋初值
        q <= 0;
    end else begin
        // 正常操作: 用 <= 非阻塞赋值
        q <= d;
    end
end
```

> **要点**: 敏感列表写 `posedge clk or negedge rst_n`（异步复位）
> 或只写 `posedge clk`（同步复位）

## 同步复位 vs 异步复位

| 特性 | 同步复位 | 异步复位 |
|------|----------|----------|
| 敏感列表 | `posedge clk` | `posedge clk or negedge rst_n` |
| 复位生效 | 需要等时钟沿 | 立即生效 |
| 优点 | 避免亚稳态, 可测性好 | 复位即时 |
| 缺点 | 需要时钟才能复位 | 可能引入毛刺 |
| 推荐 | ✅ 大部分场景 | 需要异步复位时 |
