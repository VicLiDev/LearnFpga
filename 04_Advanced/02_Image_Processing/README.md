# 02_Image_Processing — 图像处理基础

## FPGA 图像处理的优势

```
传统 CPU/GPU                    FPGA
  逐像素处理 (串行)              像素级流水线 (并行)
  内存搬运频繁                  像素数据直接流过处理单元
  灵活但延迟高                  低延迟, 确定性时序
```

## 像素数据格式

常见格式: **RGB565** (16 位, 最常用) / **RGB888** (24 位)

```
RGB565:  [R4:R0][G5:G0][B4:B0]  =  5bit R + 6bit G + 5bit B
RGB888:  [R7:R0][G7:G0][B7:B0]  =  8bit R + 8bit G + 8bit B
YCbCr:   Y=亮度, Cb/Cr=色度      (视频常用)
Gray:    8 bit                    (灰度图)
```

## 像素流处理模型

```
Camera/Sensor → [RGB→Gray] → [Gaussian] → [Sobel] → [Threshold] → Display/DMA
                  ┌─────────────────────────────────────────────┐
                  │          像素流水线 (每周期处理 1 个像素)       │
                  └─────────────────────────────────────────────┘
```

关键特点: 每个处理模块每周期处理 1 个像素, 吞吐量 = 像素时钟频率。

## 行缓冲 (Line Buffer)

Sobel 等邻域操作需要 3x3 窗口, 需要 2 条行缓冲 + 当前行的 3 个像素:

```
Line 0: [p00][p01][p02][p03]...  ← 存在 BRAM 中 (1 行)
Line 1: [p10][p11][p12][p13]...  ← 存在 BRAM 中 (1 行)
Line 2: [p20][p21][p22][p23]...  ← 当前正在输入

3x3 窗口:
  [p00][p01][p02]
  [p10][p11][p12]
  [p20][p21][p22]
```
